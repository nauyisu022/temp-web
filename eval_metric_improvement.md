# KSP 评测指标改进方案

## 问题

当前指标体系存在"数量偏差"：
- **一致率（Precision）**：生成少但精准的模型得分高，惩罚"多生成"
- **覆盖 GMV（绝对量）**：生成多的模型天然占优，奖励"多生成"
- **缺少 Recall**：无法衡量模型是否"找全了"相关搜索词

极端例子：模型只生成 1 个 KSP 且命中，一致率 100%，但毫无实用价值。

## 当前指标一览

| 指标 | 含义 | 偏向 |
|------|------|------|
| 搜索词一致率 | KSP == 搜索词 的比例 | 偏向少生成 |
| 短语命中率 | KSP ⊂ 搜索词 的比例 | 偏向少生成 |
| Fact 短语命中率 | KSP ⊂ title/desc 的比例 | 偏向少生成 |
| FactScore | KSP 拆词后在原文中的比例 | 偏向少生成 |
| 覆盖 GMV | 命中搜索词的 GMV 总和 | 偏向多生成 |
| 单词命中率 | KSP 拆词逐词匹配 | 区分度低（都 99%+） |

## 改进方案

### 方案 1：命中搜索词去重数（最直接）

每个商品覆盖了多少个**不同的搜索词**（去重后）。

- 天然平衡精准和数量：生成垃圾不加分，生成精准的多加分
- 实现简单，从现有 `top_matched_searches` 字段即可提取
- 指标：`avg_unique_search_terms_per_product`

### 方案 2：固定预算对比

每个模型只取每个商品的 **top-N 个 KSP**（如前 15 个），在相同数量下比各项指标。

- 消除"多生成"的优势
- 需要定义 KSP 的排序规则（按生成顺序？按 layer？）
- 指标：`identical_rate@15`, `phrase_hit@15`, `fact_hit@15`

### 方案 3：相对 Recall（推荐）

**核心思路**：用多个模型的并集作为"该商品相关搜索词的上限"。

对于同一个商品：
1. 收集所有模型命中的搜索词（精确匹配 + 子串匹配），取并集
2. 并集大小 = 该商品的"可发现搜索词总数"
3. 每个模型的 Recall = 自己命中的搜索词数 / 并集大小

优点：
- 不依赖外部标注，纯粹比较模型之间的相对覆盖能力
- 如果三个模型都没找到的搜索词，本来就不在评估范围内
- 可以进一步算 **F1 = 2 × Precision × Recall / (P + R)**

```
示例：商品 A
  7B    命中搜索词: {手機殼, 透明手機殼, iPhone手機殼}         → 3 个
  14B   命中搜索词: {手機殼, 防摔手機殼}                       → 2 个
  Gemini 命中搜索词: {手機殼, 透明手機殼, 防摔手機殼, 矽膠殼}  → 4 个

  并集: {手機殼, 透明手機殼, iPhone手機殼, 防摔手機殼, 矽膠殼} → 5 个

  7B     Recall = 3/5 = 60%,  Precision = 15.2%,  F1 = ...
  14B    Recall = 2/5 = 40%,  Precision = 14.2%,  F1 = ...
  Gemini Recall = 4/5 = 80%,  Precision = 13.4%,  F1 = ...
```

### 方案 4：GMV 加权指标

不是简单数 KSP 命中数，而是按命中搜索词的 GMV 加权。

- 命中 1 个高 GMV 搜索词（如"鍵盤" GMV=26.7万）比命中 10 个低 GMV 的更有价值
- 指标：`gmv_weighted_precision = 命中搜索词的 GMV / 所有可能搜索词的 GMV`
- 可以和方案 3 结合：GMV 加权的 Precision / Recall / F1

### 方案 5：分层 F1

按 Layer 分别计算，因为不同层的 KSP 性质不同：
- **Layer1（品牌品类）**：应该高精准，这层的 Precision 更重要
- **Layer2（USP）**：精准和覆盖都重要，F1 最合适
- **Layer3（长尾词）**：覆盖更重要，Recall 权重更高

## 实现优先级建议

1. **方案 1**（命中去重数）— 最简单，立刻能算，先看数据
2. **方案 3**（相对 Recall + F1）— 最有说服力，需要同商品子集数据
3. **方案 2**（固定预算）— 作为辅助验证
4. **方案 4/5** — 按需深入

## 数据依赖

- 方案 1/2：只需现有 details CSV
- 方案 3：需要同商品子集 + 每个模型的命中搜索词列表（从 `top_matched_searches` 或重新跑评测获取）
- 方案 4：需要搜索词 GMV 数据（已有）
